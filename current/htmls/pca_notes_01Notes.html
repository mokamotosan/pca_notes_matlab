諸注意事項

本稿の目的と構成

本稿の目的は「PCAをデータ分析などの日常業務で使えるようになる」ことです．そのため，以下のように構成しました．
  • PCAが適用となるデータ（第１章）
  • MATLAB statistics and machine learning toolboxをつかったPCA（第２章）
  • PCAの数学的背景として「固有値分解」を幾何学的に理解する（第３章～）
最初１章と２章はおもに，プログラミングの経験があまりない人を念頭においています．現在，PCAはコンピュータ上で比較的簡単に実行できるようになっています．統計分析フリーソフトウェア「R」や，汎用フリープログラミング言語「Python」，数値解析の商用ソフトウェア「MATLAB（ただし，Statistics and Machine Learning Toolboxが必要）」などが使える環境であれば，プログラム（「関数」）にデータを放り込むだけでPCAの解析結果を得ることが出来ます．たとえば，「MATLAB」の「Statistics and Machine Learning Toolbox」にはpca()という関数が準備されており，コマンドウィンドウに
[coefs, scores, latent] = pca(ratings,'Algorithm','eig');
と入力し，実行するだけで，PCAの主要な結果が得られます．しかし，適切に使用するにはPCAとはどのようなものかをある程度理解しておく必要があります．第１章では導入として，どのようなデータに対してPCAが有効なのかを実際のデータを使って解説します．第２章では，実際にpca()関数を使いながら，それぞれのパラメーターの意味を概説していきます．
第３章以降でPCAの数学的な背景を解説していきます．pca()関数の使い方がわかれば，さまざまなデータを自分で解析できるようになるはずです．しかし，それだけでは太刀打ちできな場合もあります．解説どおりにpca()関数を適用することは出来たけど，得られた結果やその解釈が妥当なのかどうか判断に困ることもあります．そのような場合でも，数学的な背景を理解しておくこと役に立つことがあります．第３章以降では，できるだけ視覚的なイメージでPCAの数学的な背景を解説していきます．PCAの計算方法（アルゴリズム）にはいくつかの方法がありますが，本稿では「分散共分散行列の固有値分解」を採択し，できるだけ幾何学的なイメージに基づいて解説していきます．第４章では「射影ベクトル」を導入し，第５章ではその射影ベクトルをもとに分散共分散行列を導出します．第６章以降で，PCAの数学的な背景として固有値分解の幾何学的解釈を紹介していきます．

本稿の特色

データセットを使って具体的に計算しながら説明

本稿では，実際のデータをつかって，実際にコードを動かしながらPCAの数学的な背景を説明をしていきます．こうすることで，理論と実践をなるべくスムーズに繋げられるように配慮しています．世の中には，さまざまな解析方法があり，それらの使い方に関する豊富な情報も書籍やインターネットを通じて簡単に手に入るようになりました．また，それらの数学的な背景を解説する優れた参考書も数多く存在しています．しかしながら，それらの理論と実践を繋ぐ努力は多くの場合，読者（解析者）に委ねられているように思います．そのような努力は熟達の過程で必ず求められるものですが，本稿では，理論と実践を繋ぐひとつの方法論としてなるべくたくさんの図を作って，視覚化していくというアプローチを提示します．これは私が個人的に好むやり方ですが，この方法がどなたかの参考になれば幸いです．
本稿では，MATLAB Statistics and Machine Learning Toolboxに含まれる「米国の都市における生活満足度」に関するデータを利用します．このデータはMATLABのドキュメンテーションでも，PCAの解説に用いられています（「PCA の使用による米国の都市における生活満足度の分析：https://jp.mathworks.com/help/stats/quality-of-life-in-u-s-cities.html」）．このドキュメンテーションではPCAの適用例が詳しく紹介されていますが，数学的な背景についてはほとんど触れられていません．本稿において，同じデータセットを使って数学的な背景を解説することで，読者が理論と実践を比較しながら理解を深められるように配慮しました．

グラフを多用して数学的な背景を視覚的に理解する

PCAは「データの分散をもっともよく説明する方向（ベクトル）を見つける手法」をとして説明されることがあります．そして，その視覚的なイメージとして楕円を用いた説明がよくなされます．一方，PCAの数学的な背景として「PCAはデータの分散共分散行列の固有値分解である」と説明されます．どちらもそのとおりなのですが，なぜ「データの分散共分散行列の固有値分解」が「データの分散をもっともよく説明するする方向（ベクトル）をみつける」ことになるのかについてはまとまった解説はあまり見つけられませんでした．本稿ではその「なぜ」をできるだけ視覚的なイメージを大切にしながら解説していきます．その過程で，PCAの視覚的なイメージは必ずしも楕円だけではなく，分散共分散行列の固有値分解をより正確に描写しているイメージが存在することも紹介します．

本稿は
  • プログラミングをほとんどやったことがないけど，主成分分析を出来るようになりたい
  • これまで何らかのプログラミング言語でPCAを使ってきたけど，原理を理解した上で使いたい

プログラミングに関する事前知識とリソース

プログラミングに関しては以下のようなレベルを想定しています．
  • スカラーやベクトル，行列の四則演算ができる
  • 計算結果を変数への代入ができる
  • わからないことがあればインターネットで検索できる
最初の２項目に自信がない方は，MATLABのオンラインコース（https://jp.mathworks.com/learn/tutorials/matlab-onramp.html）の受講を強くおすすめします．このコースはインターネット接続さえあればだれでも無料で受講できます．ブラウザ上で動作するため，MATLABのインストールは必要ありません．お時間のない方は最初の６セクションだけでもかまいません．これらの操作はMATLAB以外のプログラミング言語（「R」や「Python」など）でデータ解析を行う際にも基本中の基本となる操作です．また，ひとつの言語で操作方法に慣れておけば，ほかの言語でも類推して使うことが出来ます．前述のオンラインコースで一度，操作になれておいてください．
また，MATLABのヘルプやドキュメンテーションもぜひ活用してみてください．ヘルプはコマンドラインで簡単に実行できます．たとえば，pca()関数のヘルプを見る場合は，コマンドライン上に以下のように入力し，実行します．
help pca

ヘルプを実行すると，MATLABに組み込まれている関数の使い方（構文）が表示されます．関数への値の渡し方やオプションの指定方法，返ってくる出力などがわかります．もっと詳しい使い方を知りたい場合は，ドキュメンテーションが役に立ちます．ドキュメンテーションにアクセスする方法はいくつかありますが，コマンドラインから特定の関数のドキュメンテーションを開くには，下記のようにします．つ
doc pca
別ウィンドウで，指定した関数の使い方が記載されたドキュメンテーションが表示されます．ドキュメンテーションには，さきほどの構文のほかに，簡単な解説や，例などが記載されています．ヘルプやドキュメンテーションはMATLABの開発および販売元であるMathWorksにより提供されており，MATLABに関してはもっとも信頼できる一次情報です．積極的な利用を勧めます．MathWorksのインターネットサイトにはヘルプやドキュメンテーションの他に，さまざまなサポート情報が提供されています．より実践的な解説やユーザーコミュニティサイトもありますので，こちらも併せて利用されることをおすすめします.．MATLABサポートには（https://jp.mathworks.com/support.html?s_tid=gn_supp）からアクセスできます．
ヘルプやドキュメンテーションはMATLABに限らず，さまざまなプログラミング言語で用意されており，もっとも信頼できる情報リソースの一つですが，内容が専門的すぎてわかりずらい場合もあります．そうした場合はインターネット検索も役立つ場合があります．たとえば，「MATALB」と「PCA」というキーワードで検索すると，別のユーザーがまとめたサイトを見つけることができます．インターネット検索はとても強力なツールになりえます．また，ほかのプログラミング言語を使う場合にも同様の経験を活かすことができます．一方で，誤った情報があったり，環境の違いなど（たとえばMacとWindows）で，利用できないものもありますので十分に注意してください．以下に，プログラミングに関する情報交換サイトをいくつか紹介します．これらはユーザー同士が問題や解決策をシェアするサイトです．自身が抱えている問題の解決方法が見つかる場合もありますので，こちらも併せて利用されることをおすすめします．
  • stack overflow: https://stackoverflow.com/
  • Qiita: https://qiita.com/
pca()関数に限らず，プログラミングではこのようなリソースを参照しながらコードを書いていくことが日常茶飯事です．むしろ，プログラミングの必須能力だと言っても過言ではありません．プログラミングでわからないことがあった場合は自分で調べられるようにチャレンジしてみてください．

スカラー

本稿でスカラーといった場合は1,3,-5などの実数を表すこととします．不特定の数値を表す場合はローマン（）またはギリシャ文字小文字（）を用いることにします．

ベクトル

本稿でベクトルといった場合は，スカラーを縦または横に並べたものを指します．数値を縦に並べたベクトルを縦ベクトルといい，のように表します．数値を横に並べたものは横ベクトルといい，のように表します．本稿では単にベクトルといった場合は，とくに断りがない場合を除き縦ベクトルを用います．ベクトルはイタリック小文字（）で表します．ベクトルの要素を明示する場合は，のようにします．やはスカラーです．要素を縦ベクトルで表すことが難しい場合は，のように表します．ここで上付き文字のTは転置を表し，です．

位置ベクトルと基底ベクトル

ベクトルはある座標空間での位置として解釈することができます．ベクトルがある座標空間での位置を表すことを明示する場合には，そのベクトルをとくに位置ベクトルと呼びます．位置ベクトルが示す位置をグラフの中で示す場合は，座標点をローマン大文字で表します（）．また，その座標空間の座標軸を表すベクトルは特別なベクトルとして基底ベクトルと呼ぶことにします．位置ベクトルと基底ベクトルを区別したいときには，位置ベクトルをイタリック小文字．基底ベクトルをのように表します．
